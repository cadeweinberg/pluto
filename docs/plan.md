# Plan: Pluto v0.1

Pluto v0.1 is a specification-first release with one minimal executable proof slice: deterministic TI-IR text round-trip (`parse -> emit`).

## Navigation

1. [v0.1 Roadmap](plan/v0.1/roadmap.md) — ordered steps, per-step deliverables, status markers, and discussion.
2. [v0.1 Charter](plan/v0.1/charter.md) — scope boundaries, implementation baseline, and exit framing.
3. [v0.1 Terminology](plan/v0.1/terminology.md) — canonical terms and usage rules.
4. [v0.1 Decisions](plan/v0.1/decisions.md) — accepted and superseded planning decisions.
5. [v0.1 Further Considerations](plan/v0.1/further-considerations.md) — open planning questions and current direction.
6. [v0.1 Exit-Gate Checks](plan/v0.1/exit-gate-checks.md) — formal checks required to clear the exit gate.
7. [v0.1 Data Oriented Design Principles](plan/v0.1/data-oriented-design-principles.md) — architecture evaluation rubric for v0.1 data/storage decisions.

## Document Ownership

1. [v0.1 Charter](plan/v0.1/charter.md) is authoritative for scope boundaries (in/out), objective, and release form.
2. [v0.1 Roadmap](plan/v0.1/roadmap.md) is authoritative for sequencing, step status, and deliverables.
3. [v0.1 Decisions](plan/v0.1/decisions.md) is authoritative for accepted decisions and rationale.
4. [v0.1 Exit-Gate Checks](plan/v0.1/exit-gate-checks.md) is authoritative for pass/fail criteria and evidence mapping.
5. [v0.1 Further Considerations](plan/v0.1/further-considerations.md) is authoritative for unresolved questions only.
6. [v0.1 Terminology](plan/v0.1/terminology.md) is authoritative for term definitions and usage rules.
7. [v0.1 Data Oriented Design Principles](plan/v0.1/data-oriented-design-principles.md) is authoritative for architecture-evaluation rubric.

## Deconfliction Rule

If statements overlap across documents, the owning document above is the source of truth and other docs should link instead of restating normative content.
